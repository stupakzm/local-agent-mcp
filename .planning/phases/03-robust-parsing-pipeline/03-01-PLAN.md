    ```typescript

    Internal functions (NOT exported):

    1. `tryDirectParse(raw: string): unknown | null` -- JSON.parse with try/catch
    2. `stripProse(raw: string): string` -- remove markdown fences (```json, ```, ```tool_call, ```javascript, ```js), strip preamble before first `{` or `[`, strip postamble after last `}` or `]`
    3. `extractFirstJson(raw: string): string | null` -- balanced-brace depth-tracking walker (NOT greedy regex), handles nested objects, tracks `inString` and `escape` state
    4. `lenientRepair(raw: string): string` -- (a) remove trailing commas before `}` or `]`, (b) single-to-double quotes ONLY when string contains no double quotes at all, (c) strip `//` and `/* */` comments, (d) quote unquoted keys with regex `([{,]\s*)([a-zA-Z_][a-zA-Z0-9_]*)(\s*:)` -> `$1"$2"$3`
    5. `extractAllJsonObjects(raw: string): unknown[]` -- repeatedly call extractFirstJson on remaining string, parse each, collect results
    6. `normalizeToolCall(raw: unknown): { name: string; parameters: Record<string, unknown> } | null` -- (a) resolve name via NAME_ALIASES = `['name', 'tool_name', 'function', 'action', 'tool', 'function_name']`, (b) resolve parameters via PARAMS_ALIASES = `['parameters', 'arguments', 'args', 'params', 'input', 'kwargs', 'inputs']`, (c) handle double-stringified parameters (typeof val === 'string' -> JSON.parse), (d) fallback: if name found but no parameters key, collect residual keys as parameters, (e) if name still missing, call inferToolName on parameters
    7. `inferToolName(params: Record<string, unknown>): string | null` -- match against TOOL_SIGNATURES: `{ read_file: ['path'], write_file: ['path', 'content'], list_dir: ['path'], bash: ['command'] }`. Check write_file BEFORE read_file (write_file is more specific with 2 required params vs 1).
    8. `toOllamaToolCall(tc: { name: string; parameters: Record<string, unknown> }): OllamaToolCall` -- wrap into `{ function: { name, arguments: parameters } }`
    9. `extractToolCalls(raw: string): OllamaToolCall[] | null` -- run 6 strategies in pipeline order: (1) direct parse, (2) direct parse of stripProse(raw), (3) extractFirstJson -> direct parse, (4) extractFirstJson -> lenientRepair -> direct parse, (5) lenientRepair(stripProse(raw)) -> direct parse, (6) extractAllJsonObjects -> pickBestToolCall. On each successful parse, call normalizeToolCall; if normalized, return `[toOllamaToolCall(normalized)]`.
    10. `pickBestToolCall(candidates: unknown[]): { name: string; parameters: Record<string, unknown> } | null` -- prefer candidates that normalize AND have a known tool name, then fallback to first that normalizes
    11. `buildCorrectionMessages(badOutput: string, parseError: string): OllamaMessage[]` -- return a single user message with the correction prompt template from PARSING.md section 3.3. Template: "Your previous response could not be parsed as a tool call.\n\nHere is what you returned:\n<previous_response>\n{badOutput}\n</previous_response>\n\nThe parsing error was: {parseError}\n\nIMPORTANT: Your response must be ONLY a JSON object with no other text. Required format:\n{\"name\": \"<tool_name>\", \"parameters\": {<key>: <value>}}\n\nDo not include any explanation, preamble, or text outside the JSON object."
    12. `getParseError(content: string): string` -- describe why parsing failed (e.g., "No JSON object found in response", "JSON parsed but missing tool name", "Truncated JSON -- no closing brace")

    Main `parseToolCall` function logic (per D-01, D-02):
    ```
    1. Try extractToolCalls(content)
    2. If success, return result
    3. If fail, enter retry loop (max 3 iterations per D-04):
       a. Build correction messages from (rawContent, lastError)
       b. Call chatFn(correctionMessages)
       c. Update rawContent = response.content
       d. Try extractToolCalls(response.content)
       e. If success, return result
       f. Update lastError = getParseError(response.content)
    4. After 3 retries, return ParseFailure { reason: lastError, rawContent, attemptCount: 3, lastError }
    ```

    All logging via `console.error` only. Never `console.log`. Never throw -- always return ParseFailure.
