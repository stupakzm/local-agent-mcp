  <action>
    Create `src/security.ts` as a standalone module with NO external dependencies (only `node:path` and `node:process`).

    **1. Path validation (SAFE-01):**
    ```typescript
    export function assertPathSafe(targetPath: string, root: string): string
    ```
    - Resolve targetPath against root: `const resolved = path.resolve(root, targetPath)`
    - Check prefix: `if (!resolved.startsWith(root + path.sep) && resolved !== root)` then throw
    - The `root + path.sep` check prevents `/project-evil/` matching `/project` (per SAFE-01 requirement)
    - Return the resolved absolute path on success
    - Error message: `"path not allowed (use AGENT_ALLOWED_PATHS to grant access)"`

    **2. Command allow-list (SAFE-02, SAFE-03):**
    ```typescript
    export function assertCommandAllowed(command: string, allowList: readonly string[]): void
    ```
    - Trim the command, extract first token: `const firstToken = trimmed.split(/\s+/)[0]`
    - Check: `if (!allowList.includes(firstToken))` then throw
    - This naturally handles the prefix attack: `gitevil` is not in the list, only `git` is
    - Error message: `"command not allowed (use AGENT_ALLOWED_COMMANDS to add)"`

    **3. Default allow-list (SAFE-03):**
    ```typescript
    export const DEFAULT_ALLOWED_COMMANDS: readonly string[] = [
      "git", "ls", "cat", "echo", "grep", "find", "mkdir", "cp", "mv", "touch", "npm", "node", "python"
    ] as const;
    ```
    Explicitly excludes: `rm`, `curl`, `wget`, `sudo`, `ssh`.

    **4. Safe subprocess environment (SAFE-05):**
    ```typescript
    export const ALLOWED_ENV_KEYS = ["PATH", "HOME", "USER", "LANG"] as const;
    export function buildSafeEnv(): Record<string, string>
    ```
    - Iterate ALLOWED_ENV_KEYS, pick from process.env, skip undefined
    - Returns a fresh object -- no reference to process.env

    **5. Output truncation (SAFE-06):**
    ```typescript
    export const MAX_OUTPUT_BYTES = 1_048_576; // 1MB
    export function truncateOutput(output: string): string
    ```
    - If `Buffer.byteLength(output) <= MAX_OUTPUT_BYTES` return as-is
    - Otherwise: slice to fit within byte limit, append `\n[output truncated: exceeded 1MB limit]`

    **6. Shell mode types (SAFE-07):**
    ```typescript
    export type ShellMode = "restricted" | "full" | "none";
    export function isShellModeRestricted(mode: ShellMode): boolean
    ```
    - Returns `mode === "restricted"`

    **7. Add vitest to devDependencies in package.json:**
    - Add `"vitest": "^3.1.0"` to devDependencies
    - Add `"test": "vitest run"` to scripts
    - Add `"test:watch": "vitest"` to scripts

    **8. Update tsconfig.json:**
    - Add `"src/__tests__"` is already covered by `"include": ["src/**/*"]` -- no change needed unless tests use special types
    - Verify `include` pattern covers `src/__tests__/**/*.test.ts` (it does via `src/**/*`)

    All functions must be pure (no side effects except reading process.env in buildSafeEnv). All must use `export` for ESM. No `console.log` anywhere -- only `console.error` if logging is needed (but these are pure functions, no logging expected).
