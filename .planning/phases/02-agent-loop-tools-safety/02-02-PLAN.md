    **Types:**
    ```typescript
    export interface OllamaMessage {
      role: "system" | "user" | "assistant" | "tool";
      content: string;
      tool_calls?: OllamaToolCall[];
    }

    export interface OllamaToolCall {
      function: {
        name: string;
        arguments: Record<string, unknown>;  // Pre-parsed object, NOT a JSON string
      };
    }

    export interface OllamaToolDefinition {
      type: "function";
      function: {
        name: string;
        description: string;
        parameters: {
          type: "object";
          properties: Record<string, { type: string; description: string }>;
          required: string[];
        };
      };
    }

    export interface OllamaChatRequest {
      model: string;
      messages: OllamaMessage[];
      tools?: OllamaToolDefinition[];
      stream: false;
    }

    export interface OllamaChatResponse {
      message: OllamaMessage;
      done: boolean;
    }
    ```

    **Client function:**
    ```typescript
    export async function chatWithOllama(
      host: string,
      request: OllamaChatRequest
    ): Promise<OllamaChatResponse>
    ```

    Implementation:
    - `const url = \`\${host}/api/chat\``
    - `const resp = await fetch(url, { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify(request) })`
    - If `!resp.ok`: throw with `Ollama error: ${resp.status} ${resp.statusText}`
    - Parse response as JSON, return typed
    - Wrap fetch in try/catch: if fetch itself throws (ECONNREFUSED), throw `"Ollama is not running at ${host} -- start it with: ollama serve"`
    - `stream: false` is always set in the request type -- streaming is out of scope

    **Critical gotcha from ROADMAP:** `arguments` in tool_calls is a pre-parsed object, NOT a JSON string. The type `Record<string, unknown>` is correct -- do NOT JSON.parse it.

    No console.log. If any debug logging needed, use console.error.

    **Tool executor dispatch:**
    ```typescript
    export interface ToolResult {
      success: boolean;
      output: string;
    }

    export async function executeTool(
      name: string,
      args: Record<string, unknown>,
      workingDir: string,
      shellMode: ShellMode,
      allowedCommands: readonly string[],
      timeoutMs: number
    ): Promise<ToolResult>
    ```

    Dispatch by name to internal functions. Unknown tool name returns `{ success: false, output: "unknown tool: {name}" }`.

    **read_file (TOOL-01):**
    - Extract `path` from args as string
    - `const safePath = assertPathSafe(path, workingDir)` -- throws if outside root
    - `const content = await fs.readFile(safePath, "utf-8")`
    - Return `{ success: true, output: content }`
    - On error: return `{ success: false, output: error.message }`

    **write_file (TOOL-02):**
    - Extract `path` and `content` from args
    - `const safePath = assertPathSafe(path, workingDir)` -- throws if outside root
    - `await fs.mkdir(path.dirname(safePath), { recursive: true })` -- mkdir -p semantics per CONTEXT.md
    - `await fs.writeFile(safePath, content, "utf-8")`
    - Return `{ success: true, output: "wrote {byteLength} bytes to {path}" }`

    **list_dir (TOOL-03):**
    - Extract `path` from args
    - `const safePath = assertPathSafe(path, workingDir)`
    - `const entries = await fs.readdir(safePath, { withFileTypes: true })`
    - For each entry, `await fs.stat()` to get size and mtime
    - Format as `ls -la` style per CONTEXT.md: `"{type} {name} {size}B {modified_iso_date}"` per line
    - Types: `d` for directory, `f` for file
    - Return `{ success: true, output: formattedList }`

    **bash (TOOL-04):**
    - If `shellMode === "none"`: return `{ success: false, output: "bash is disabled (shell mode: none)" }`
    - If `shellMode === "restricted"`: `assertCommandAllowed(command, allowedCommands)` -- throws if blocked
    - Use `child_process.execFile("bash", ["-c", command], options)` with:
      - `cwd: workingDir`
      - `timeout: timeoutMs`
      - `env: buildSafeEnv()` -- restricted env per SAFE-05
      - `maxBuffer: MAX_OUTPUT_BYTES + 1024` -- slightly over cap to detect truncation
      - On Unix: `detached: true` for process group kill -- set up `kill(-child.pid, SIGTERM)` on timeout (SAFE-04)
    - Combine stdout + stderr in output
    - Apply `truncateOutput()` to result (SAFE-06)
    - If `shellMode === "full"`: append `"\n[shell mode: full — no restrictions applied]"` to output per CONTEXT.md
    - On timeout error: return `{ success: false, output: "command timed out after {seconds}s" }`
    - On other error: return `{ success: false, output: error.message }`

    Wrap ALL tool execution in try/catch -- security assertion throws become `{ success: false, output: error.message }`. Never let a throw escape executeTool.

    Import from `node:fs/promises`, `node:path`, `node:child_process`. Import security functions from `./security.js`.
